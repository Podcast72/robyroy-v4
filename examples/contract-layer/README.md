# Contract Layer

Robyroy-V4 uses contract files to make the operating model easier to inspect.

Example concepts:

- `skill.contract.json` identifies the skill and modes.
- `capabilities.json` declares allowed capabilities and blocked actions.
- `failure_codes.json` maps stop conditions to stable codes.
- `report_schema.json` defines the final receipt format.

A contract layer makes the skill more repeatable than a free-form prompt alone.
