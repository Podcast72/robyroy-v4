# Basic Usage

Use Robyroy-V4 when an agent task needs a clear operating frame.

```text
$robyroy-v4
OBJECTIVE: Update this README section.
CONSTRAINTS: no commit, no push, docs only.
```

Expected behavior:

- detect the mode;
- define or confirm the Goal Contract;
- identify allowed and forbidden paths;
- apply compact Karpathy-style discipline before editing;
- keep changes simple and surgical;
- run local checks where available;
- declare lifecycle statuses in the final receipt;
- produce evidence and limits.

Example lifecycle status fields:

```text
V4_SESSION_STATUS:
EVENT_LOGGING_STATUS:
V4_TRACE_STATUS:
HANDOFF_STATUS:
PROMPT_CHAIN_STATUS:
RUNNER_STATUS:
REPORT_CHECK_STATUS:
IMPROVEMENT_CANDIDATE_STATUS:
```
