# Security Policy

Robyroy-V4 is an experimental local workflow skill. It is not a security boundary by itself.

## Reporting issues

If you find a security-relevant issue, open a private report through the repository owner's preferred contact channel or create an issue that does not include sensitive material.

## Sensitive input

Do not place sensitive values, credentials, private keys or environment files into examples, logs, issues or pull requests.

## Expected handling

Reports should include:

- a concise description
- affected files or behaviors
- reproduction steps when safe
- suggested mitigation if known

Keep reports evidence-based and avoid publishing sensitive data.
